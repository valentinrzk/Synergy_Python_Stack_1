## Структура репозитория

Для удобства все задания организованы в одном репозитории и распределены по папкам.

В папке `src` находится каталог с директориями, соответствующими номерам уроков. Некоторые из номеров пропущены, т.к. не для всех уроков предусмотрены практические задания.
Внутри папок `lesson_*` содержатся файлы `*.py`, содержащие исходный код выполненных задач, а также файл `task_description.md`, содержащий описание задач.

В папке `tests` содержатся тесты к задачам из каждого урока. Каталоги организованы аналогично структуре папки `src`. Внутри каталога есть файлы `test_*.py`, содержащие код тестов, а также `cases.py`, содержащие набор тест-кейсов для параметризации тестов.

## Подготовка к запуску

После клонирования репозитория необходимо подготовить виртуальное окружение и установить зависимости.

- Если вы используете `uv` (рекомендуется), выполните в терминале команду `uv sync`
- Если вы используете `pip`, на такой случай в репозитории подготовлен файл `requirements.txt`. Выполните команду `pip install -r requirements.txt`

## Запуск

### Запуск кода

Для запуска  кода выполните команду `python src/lesson_*/task_*.py`, либо запустите файл любым иным удобным для вас способом.

### Запуск тестов

Для запуска тестов используйте следующие команды:

1. `pytest` - запуск всех тестов;
2. `pytest -v` запуск всех тестов c подробной информацией;
3. `pytest -v tests/tests_lesson_* ` запуск тестов по конкретному уроку.

### Запуск проверок Code Quality

Для обеспечения code quality используется `ruff`(линтер и форматер) и `ty` (статический анализатор). Команды:
1. `ruff check .` - запуск линтера;
2. `ruff format . --check` - запуск форматтера;
3. `ty check` - запуск статического анализатора
